Необходимо реализовать приложение используя:

Java
Spring Boot
build- и SCM- tools (Maven/Gradle, Git)
Docker для контейнеризации приложения
PostgreSQL, либо другая СУБД на выбор.
Плюсом будет:

Helm chart для возможности запуска приложения в k8s/minikube
JUnit, Mockito, Testcontainers.
 
Требования к приложению:

Написать два микросервиса, расположенных в отдельных контейнерах и выполняющие следующие функции:

Микросервис Клиент

Отправка сообщений по API
Реализация должна быть выполнена по REST API, плюсом будет если соискатель покажет знания работы с GRPC.

Микросервис Сервер

Прием сообщений от микросервиса Клиент.
Поток должен уметь считывать сообщения и писать их в базу в виде минимального логирования – id сообщения, дата/время, текст сообщения.

Реализация должна быть выполнена на базе REST, плюсом будет если соискатель продемонстрирует навыки работы с gRPC. Для метода необходимо реализовать интеграционные тесты.

 
Критерии приемки:

1. В приложении REST API/ GRPC API реализовано два метода:

отправить сообщения
вернуть записи из БД БД в соответствии с параметрами: id сообщения, период дат с/по.
2. Наличие интеграционных тестов.

 
Результаты работы опубликовать на GitHub, либо Bitbucket вместе с короткой инструкцией по сборке, развертыванию и использованию.
